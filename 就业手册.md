# 1.什么是Java的多态特性？

Java的多态性是指同一个方法作用在不同的类上可以有不同的实现，并根据实际引用的对象类型执行相应的方法。

**底层原理：编译看左边，运行看右边**

多态的三个必要条件：

- 要有继承或者实现关系，子类继承父类或者类实现接口
- 子类要重写override父类的方法
- 父类引用指向子类对象，也叫向上转型

多态的意义：多态其实是一种抽象行为，它的主要作用是让程序员可以面对抽象编程而不是具体的实现类，这样写出来的代码拓展性会更强。

编译时多态和运行时多态是面向对象编程中的两种实现方式，他们分别在不同的阶段决定方法的绑定。

编译时多态也叫静态多态，实现方式是方法重载

运行时多态也叫动态多态，实现方式是方法重写

# 2.Java的优势是什么？

四大优势：跨平台、内存自动管理、生态强大、工程化友好

![image-20260304160032053](/image-20260304160032053.png)

- 跨平台：代码编译成字节码后，JVM负责编译成不同操作系统的机器指令。同一套代码在windows开发不需要改一行代码就可以拿到Linux系统去运行使用。
- 内存自动管理：Java自带垃圾回收机制GC,程序员不需要手动去申请和释放内存，只需要专心处理业务逻辑即可，极大地提高了开发效率。
- 生态强：Java生态圈比较庞大，有很多现成的、成熟的开源库（如Spring全家桶）。使用起来很方便。
- 工程化友好：Java本身具有面向对象封装、继承、多态等特性，代码结构清晰，适合多人协作开发的大型项目。

Java的劣势：

- 启动慢：JVM加载。JIT编译都需要时间。
- 内存占用高:JVM本身就要几百MB，再加上堆内存，一个简单的应用就要500MB起步。
- 语法啰嗦：一个POJO要写一堆getter和setter，虽然有lombok缓解，但是比之Kotlin和data class还是太繁琐。

# 3.Java中的参数传递是按值还是按引用？

Java中的参数传递都是按值传递的（Java从设计之初就追求简洁与安全，引用传递会让代码行为变的不可预测），不论是基本类型还是引用类型。

- 基本类型数据如int、char、boolean等（存储在栈内存中）：传递的是指的副本，也就是基本类型的数据本身。对方法参数的任何修改都不会影响到原始变量。
- 引用数据类型（存储的是对象在堆内存中的地址）如对象引用（数组、对象）：传递的是引用的副本，也就是对应引用的内存地址。方法内可以通过引用修改对象的属性，但不能改变引用本身，使其指向另一个对象。

# 4.为什么Java不支持多重继承？

核心原因是菱形继承问题。吸取C++多继承带来的经验，Java在语言层面上禁止多继承。

![image-20260304171717088](/image-20260304171717088.png)

C++允许多继承，但是多继承的这种歧义问题必须有程序员来处理，一不小心就会有bug，Java索性一刀切，禁止类的多继承。

# 5.Java面向对象编程与面向对象编程的区别是什么？

**面向对象编程**是将类或对象作为基本单元来组织代码，围绕封装、多态和继承来设计。**面向过程编程**是以过程或函数为中心，对应到代码中就是函数，函数和数据是分离的。

简单来说，面向对象注重对象之间的交互和模块化设计，面向过程注重逻辑的分步实现。

主要区别：

1. 思维方式不同：面向对象通过定义对象的属性和行为去解决问题，关注对象之间的关系和交互。面向过程通过函数一步步实现业务逻辑，注重执行的步骤和顺序。

2. 数据和行为的关系不同：面向对象是将数据和行为封装在对象中，数据操作由对象方法管理。面向过程数据和函数是分开的，数据由函数操作。

3. 可扩展性和复用性不同：面向对象通过继承、多态和封装等机制支持高复用性和扩展性。面向过程复用性比较低，扩展需要修改已有代码，影响稳定性。

4. 适用场景不同：面向对象适合处理复杂的系统和模块化设计，便于扩展和维护。面向过程适合处理简单、顺序性强的程序，开发效率比较高。

![image-20260306104852406](/image-20260306104852406.png)

面向对象的三大特性：封装、继承、多态。

# 6.Java方法重载和方法重写之间的区别是什么？

**方法重载**发生在同一个类中，允许有多个同名方法，只要参数列表不同就行，参数个数、顺序、类型不同都算。主要用于在同一类中定义不同场景下的行为。

**方法重写**发生在继承关系中，子类继承父类的某个方法，参数列表和方法名必须与父类一致，从而为该方法提供不同的实现。主要用于实现运行时多态。

核心区别：

| 维度       | 重载Overloading | 重写Overriding                                               |
| ---------- | --------------- | ------------------------------------------------------------ |
| 发生位置   | 同一个类中      | 子类和父类之间                                               |
| 参数列表   | 必须不同        | 必须相同                                                     |
| 返回类型   | 可以不同        | 必须相同或者是父类返回类型的子类                             |
| 访问修饰符 | 不受限制        | 不能比父类更严格                                             |
| 静态方法   | 可以重载        | 不能重写，只能隐藏                                           |
| 异常处理   | 不受限制        | 不能抛出比父类更多的异常（可以抛出与父类一致的异常或者父类异常的子类） |

# 7.什么是Java内部类？它有什么作用？

Java内部类就是定义在另一个类里面的类，分为成员内部类、静态内部类、局部内部类和匿名内部类，内部类最大的特点是可以直接访问外部类的成员，包括私有成员。

内部类的作用：

- 把逻辑相关的类放在一起，提高内聚性，外面用不到的类就不用暴露出去。
- 内部类可以直接拿到外部类的成员变量和方法，使用起来很方便。
- 对于只在一个地方使用的小类，使用内部类可以减少代码冗余。
- 匿名内部类事件监听、回调这些场景用的非常多，如Android点击事件、swing的按钮监听，写起来很简洁。

四种内部类的区别和用法：

![image-20260308100455234](D:\note\img\image-20260308100455234.png)

成员内部类：定义在类里面但是不用static修饰，可以访问外部类的所有成员变量，包括private的。每个成员内部类实例都隐式持有一个外部类实例的引用。

静态内部类：使用static修饰，只能访问外部类的静态成员。它本质上就是个顶级类，可以独立于外部类使用，主要用来表明类结构和命名空间。

局部内部类：定义在方法里面，只在这个方法内可见，他可以访问外部类的成员，也可以访问方法中的局部变量，但是这个局部变量必须是final或者effectively final的。实际开发中用的比较少。

匿名内部类：没有类名，在创建对象的时候直接定义。用得最多的场景就是实现接口和实现抽象类。

# 8.Java中String、StringBuffer和StringBuilder的区别是什么？

三者都用于处理字符串，区别在于可变性和线程安全。

String是不可变的。底层的char数组被final修饰，一旦创建就定死在那里。每次拼接、替换都会生成新的对象，原来那个不会变。

StringBuffer和StringBuilder是可变的。底层使用可扩容的数组存字符。区别在于StringBuffer的方法都加了synchronized，是线程安全的，StringBuilder没加锁，单线程下性能最好。

实际选型：

- 字符串基本不变，或者只是少量拼接，直接用String
- 多线程下需要频繁多次拼接修改字符串使用StringBuffer
- 单线程下大量拼接操作，使用StringBuilder

![image-20260308105922272](D:\note\img\image-20260308105922272.png)

**jdk9把String、StringBuffer、StringBuilder底层的char数组都换成了byte数组。**

**Java8编译器对字符串拼接的优化是将字符串拼接转换为StringBuilder操作**。

# 9.Java的StringBuilder是怎么实现的？

StringBuilder底层就是一个可扩容的字符串数组加一个count变量。

核心数据结构很简单，继承自AbstractStringBuilder

1. append操作的流程：
   先计算一下要追加的内容需要占几位
2. 检查一下当前数组剩余容量够不够追加内容占用，不够就扩容
3. 把内容拷贝到数组末尾
4. 最后更新count值

**扩容策略是将原容量乘2再加2.扩容的本质是Arras.copyOf，创建新数组把旧数据拷贝过去，所以开销还是比较大的，建议创建StringBuilder是提前预估最终长度。**

StringBuilder提供append()，replace(),charAt(),substring()等方法。toUpperCase()属于字符串语义转换方法，只存在于String和相关工具类中，用于生成新的字符串。
